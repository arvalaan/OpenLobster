# Memory

The Memory view gives you direct access to the agent's knowledge graph — the structured store of facts, entities, and relationships the agent builds over time as it has conversations and processes information.

The view is divided into two panels:

* **Left sidebar** — A searchable index of all memory nodes, grouped by type.
* **Right panel** — Details for the selected node, including its properties, connections, and available actions.

A graph visualization tab lets you explore the relationships between nodes visually.

## When to use the Memory view

* Review what the agent has learned about a specific user, topic, or entity.
* Correct inaccurate or outdated information stored in the graph.
* Delete nodes that should no longer be part of the agent's knowledge.
* Explore relationships between entities to understand how the agent's knowledge is structured.

{% hint style="info" %}
The agent builds its memory automatically as it handles conversations. You do not need to add entries manually under normal operation.
{% endhint %}
