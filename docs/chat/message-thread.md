# Message Thread

The message thread is the main panel of the Chat view. When you select a conversation from the list on the left, the full history of that exchange loads here in chronological order, oldest messages at the top and newest at the bottom.

## Message types

Each message in the thread carries a label that identifies its origin and where it came from in the message processing pipeline:

| Label | Origin | Where it comes from | What it tells you |
| ----- | ------ | ------------------- | ------------------|
| **USER\_xxxx** | Message from a user | Step 1 of the [message processing pipeline](../architecture/processing-messages.md): Channel Adapter receives it. Suffix is conversation ID last 4 chars. | User intent, questions, or requests. The raw input. |
| **OPENLOBSTER** | Reply from the agent | Steps 3-8 of the pipeline: Agent processes context, calls AI provider, generates response. Step 11 returns it. | Agent's answer after considering history, tools, and memory. |
| **TOOL** | Record of a tool call | Step 9 of the pipeline: Agent needed extra information and executed a tool. | What the agent did behind the scenes to get data (web search, database query, file read, etc.). Shows input and output. |
| **SYSTEM** | Automated platform event | Generated by the channel adapter or session manager. Examples: "user joined group", "session started", "pairing requested". | Context about conversation lifecycle, not a message from user or agent. |

Every message displays a **timestamp** showing when it was sent/recorded. You can use timestamps to calculate latency:
- **USER timestamp → OPENLOBSTER timestamp** = How long it took agent to respond (includes steps 3-11 of pipeline)
- If timestamps are far apart, check Recent Logs for which step is slow (AI provider? Tool execution?)

## Message formatting

The thread renders rich formatting so that the agent's responses are easy to read:

* Bold and italic text
* Ordered and unordered lists
* Inline code and code blocks with syntax highlighting
* Clickable links

## Tool call messages

**TOOL** messages give you visibility into what the agent did behind the scenes. For example, if the agent searched the web or queried a database before answering, you will see a TOOL entry showing which tool was called and what it returned.

{% hint style="info" %}
Tool call messages are for operator visibility only. Users on the messaging platform do not see them.
{% endhint %}

## Group conversations

In group conversations, each **USER\_xxxx** label uses the last four characters of the conversation ID rather than the participant's name. The agent's replies are labelled **OPENLOBSTER** regardless of whether the conversation is direct or in a group.
